# DanceClip AI MVP PRD

## 1. 产品定位

DanceClip AI 是面向 TikTok / Reels / Shorts 创作者的 AI 跳舞短视频生成工具。用户上传一张清晰单人真人照片，选择低风险预设舞蹈模板，生成约 5 秒无声短视频，用于内容测试、娱乐素材和社媒发布。

MVP 不验证完整订阅 SaaS，而是验证：搜索流量用户是否能在可接受的生成质量、安全风险和单位成本下，完成一次满意生成，并为高清/去水印下载付费。

## 2. MVP 验证目标

核心假设：

- 用户愿意登录后上传真人照片生成舞蹈视频
- 模型能稳定产出可发布的 5 秒短视频
- 安全审核能拦截高风险输入和输出
- 用户愿意为单个满意结果付费解锁高清/去水印
- 单次生成成本、失败率和审核成本可支撑商业化

成功指标：

- 技术成功率：返回可播放 MP4 并成功转存，目标 > 95%
- 安全通过率：合规输入下结果通过审核，目标 > 90%
- 可用成片率：人工抽检或用户下载行为判断，目标 > 60%
- 登录后首次生成完成率：目标 > 60%
- 满意结果付费解锁率：上线后实测，不预设虚假乐观目标

## 3. 首版页面范围

公开页面：

- `/`
- `/ai-dance-generator`
- `/ai-twerk-generator`
- `/pricing`
- `/terms`

`/ai-twerk-generator` 首版只作为 SEO/需求验证页，不开放公开生成。页面可展示非露骨样例、合规说明、等待名单或跳转普通舞蹈生成器。

## 4. 用户流程

匿名用户可以：

- 浏览生成器
- 上传图片并进行输入预审核
- 选择模板和画幅
- 查看样例

点击生成时要求 Google 登录。登录后赠送 1 次免费低清带水印生成。

生成成功后：

- 用户可预览低清带水印结果
- 高清/去水印下载需要单视频付费解锁
- 高频用户可升级为一个月付订阅档

MVP 不做完整历史页。结果页提供短期访问和限时下载链接。

## 5. 付费模型

MVP 只保留两个付费路径：

- 单视频解锁：高清 + 去水印下载，价格待成本评测后确定，建议测试区间 `$2.99-$4.99`
- 一个订阅档：每月固定生成次数 + 高清/去水印权益，价格待成本评测后确定，建议测试区间 `$9.99-$19.99/月`

MVP 不做：

- 三档订阅
- 年付
- Ultra
- 公开 credits
- 一次性 credit 包

权益模型使用：

- 生成次数：一次任务消耗 1 次
- 下载解锁：某个成功视频是否已解锁高清/去水印

模型失败、系统超时、结果安全失败时，自动返还生成次数。

## 6. 上传规则

MVP 采用保守审核策略：高不确定性默认拒绝。

允许：

- 清晰单人真人照片
- 看起来明确成年
- 非裸露、非色情、非明显性化姿势
- 用户确认拥有合法使用权

拒绝：

- 未成年人或疑似未成年人
- 无法判断年龄
- 裸露、内衣、泳装、色情或明显性化姿势
- 多人图、群像、非人物图
- 动漫/虚拟人物/AI 角色
- 名人、公众人物、网红等疑似未经授权人物
- 脸部严重遮挡
- 低清到无法判断主体
- 缺少可用于驱动的人体结构
- 其他高风险或不确定输入

用户生成前必须确认：拥有图片使用权，且不得上传他人、未成年人、公众人物或未经授权照片。

## 7. 模板范围

首版公开生成只提供低风险舞蹈模板，例如：

- Hip-hop
- K-pop style
- Shuffle
- Salsa
- Robot dance
- Fitness dance
- Party dance
- Catwalk

每个模板需要：

- 封面
- 5 秒样例
- 风险等级
- 平台标签
- 模型参数
- 来源记录

`twerk` 模板只进入内部测试，不进入公开 MVP。

MVP 完全不支持 BGM，只输出无声视频。用户在 TikTok / Reels / Shorts 发布时自行添加平台授权音乐。

## 8. 视频模型与评测

首选候选：Seedance 1.0 Pro Fast via EvoLink。

MVP 代码只接入 1 个视频模型供应商，但必须通过 `ModelProvider` 抽象封装，不允许把供应商 API 直接写死进业务逻辑。

任务需记录：

- provider
- model
- providerJobId
- payload version
- cost
- latency
- failure reason

上线前至少评测 2 个候选供应商，比较质量、成本、延迟、稳定性、内容政策和数据条款。

## 9. 任务状态机

生成链路必须是幂等、可恢复的状态机。

推荐状态：

- `created`
- `input_reviewing`
- `rejected`
- `reserved`
- `submitted`
- `processing`
- `transferring`
- `output_reviewing`
- `succeeded`
- `failed_refunded`
- `blocked_refunded`

必须支持：

- idempotency key
- callback 重放保护
- Cron 兜底轮询
- R2 转存失败恢复
- 超时释放权益
- 统一状态转移函数
- 生成次数预占 + 最终确认/释放

## 10. 内容安全

三层审核：

- 上传图审核
- 请求审核：只能选择已审核模板
- 结果审核：完成后抽帧/内容检查

结果审核不通过时：

- 不展示视频预览
- 不提供下载
- 不展示具体违规帧
- 自动返还生成次数
- 前台只显示通用安全失败提示

后台短期保留最小必要审核证据，用于复查和安全审计。

## 11. 存储与隐私

匿名任务不生成正式视频。登录用户生成结果 MVP 默认保留 7-14 天。付费解锁视频提供限时下载链接。

用户删除任务后，应删除：

- 上传原图
- 生成视频
- 可公开访问链接
- 相关 R2 对象引用

审核证据、支付记录、风控日志按最小必要原则短期保留。

隐私承诺：

> 产品侧不使用用户上传图片或生成结果训练模型，也不公开展示用户内容，除非用户主动授权。第三方模型供应商的数据保留、训练使用、人工审核和删除能力，必须上线前完成条款审查。

## 12. 登录与支付

登录：

- MVP 只支持 Google 登录

支付：

- 使用 Creem
- webhook 是订阅和购买状态事实来源
- 所有 webhook 必须幂等
- 购买、订阅、退款、补偿都必须写入权益流水
- 拒付或明显滥用账号可冻结，并保留申诉入口

## 13. 内部后台

MVP 后台只服务排障、风控和手动干预，不做完整运营系统。

需要支持：

- 任务查询
- 用户查询
- 支付/权益流水查询
- 手动补偿
- 任务重试
- 用户封禁
- 模板启用/禁用
- 风险模板下架
- 管理员操作审计

后台不进入公开导航、sitemap 或 SEO。仅允许管理员 allowlist 访问，关键操作必须审计。

## 14. 投诉与滥用处理

MVP 提供邮箱或简单表单作为投诉/删除入口。

被投诉内容：

- 立即冻结下载
- 进入人工复核
- 确认违规后删除内容
- 重复违规账号封禁

下载页和法律页面应提供清晰的投诉/删除入口。

## 15. Launch Blockers

以下事项未完成不得上线：

- 完成 100-300 张真实样本的模型质量/成本/延迟评测
- 至少评测 2 个视频模型供应商
- 完成主供应商数据处理条款审查
- 上传审核可拒绝未成年人、疑似未成年人、裸露、多人、非真人、高不确定性输入
- 结果审核失败时不展示预览并返还生成次数
- 任务状态机支持幂等 callback、转存失败恢复、超时释放权益
- 支付 webhook 幂等，权益发放有流水
- 用户删除可删除原图和结果视频
- 投诉/删除入口可用
- 后台可查任务、补偿、封禁、下架模板
- 公开 MVP 不开放 twerk 生成

## 16. 明确不做

MVP 不做：

- 自由 prompt
- 用户上传参考舞蹈视频
- twerk 公开生成
- BGM
- 公开作品画廊
- 完整历史页
- 三档订阅
- 年付
- Ultra
- credit 包
- 邮箱/密码登录
- 完整投诉工单系统
- 生成完成邮件通知
- 社媒一键发布
- 团队账号
- 移动 App
- 中文 SEO
